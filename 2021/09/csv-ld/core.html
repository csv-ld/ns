formatVersion
The version of the CSV-LD format that a processor should assume for this spreadsheet.
vocabBase
The "vocabulary base" for terms in this spreadsheet.
prefix
Defines a prefix so that common URI bases can be prefixes of term URIs.
id
The URI for this sheet.
csvw
The URI for JSON-LD CSVW metadata for this sheet.
type
A category assigned to a subject. a subject may have multiple types.
label
A human-readable name for the subject.
comment
A description of the subject resource.
line
An identifier for the line of a changesheet. Useful if you want to annotate the change represented by that one line.
block
An identifier for a block, a set of lines, of a changesheet. Useful if you want to annotate the change represented by the block. A block is analogous to a git commit or database transaction.
in or out
Is this line being asserted ('in') or retracted ('out')? If left blank, assumes asserted.
namespace
Do you want to add this to the "main" namespace? A "test" namespace. If left blank, "main" is assumed. Analogous to a named graph or git branch.
subject
An identifier for the subject of the statement to be asserted or retracted. If left blank and if previous row's object cell is also blank, then an identifier is generated for both. If not in the form of a URL or CURIE (using a registered #prefix), then the cell value is interpreted to be a variable so as to unify with its usages in other cells in the changesheet -- an identifier is generated to substitute for the variable.
predicate
An identifier or recognized label for the predicate/attribute part of the statement.
object
An identified or literal value for the object/value part of the statement. If left blank and if next row's subject cell is also blank, then an identifier is generated for both. If the value is the same as that of a variable introduced earlier in the current sheet for a statement's subject, then the value is interpreted as an instance of that variable, rather than as a literal string value with the same sequence of characters.
